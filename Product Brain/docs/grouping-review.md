# 实验分组页 Review

**File:** `02-Grouping.html`
**Date:** 2026-06-22
**Product context:** Experiment R&D Platform — 实验分组（场长上传分组信息）

---

## 总评

实验分组页遵循了所有内页的通用布局（实验信息头 + 阶段流程 + 内容区），流程节点状态可视化符合设计系统规范。但作为"场长上传分组"这一关键步骤，页面缺少上传确认流程和数据校验，且与 `experiment-detail.html` 中的分组模块功能重叠。

---

## 发现清单

### P0

**1. 同 experiment-detail.html 的分组模块存在功能冗余**

两个页面实现了几乎相同的分组功能，但表单字段、交互方式不同。这会使用户困惑——"我该在哪分组？"

> **建议：** 明确分工：`02-Grouping.html` 作为独立页面（从概览页→新建实验→跳转至此），`experiment-detail.html` 的分组模块作为工作台快速入口。确保两者字段一致。

**2. 缺少分组数据校验**

分组表格的"完成分组"按钮直接标记完成，不检查处理组是否分配完整、栏位号是否重复、初始头数是否合理。

> **建议：** 完成分组前校验：所有处理组至少分配一个栏位、栏位号唯一性、初始头数 > 0。

### P1

**3. 流程节点"实验分组"应标记为 active，但视觉仅为可点击**

当前页面的流程节点无 `currentStep` / `active` 状态关联——所有节点显示为默认样式。用户不清楚自己在流程中的位置。

**4. 下载分组模板生成 CSV 但无导入功能**

`downloadTemplate` 可下载 CSV 模板，但缺少相应的导入/解析 CSV 功能。上传按钮仅 mock 添加 3 行数据。

**5. 缺少"返回概览"导航**

页面没有返回概览页的导航按钮。用户完成分组后不知道下一步去哪。

### P2

**6. 分组表格列宽固定，部分列过于窄小**

处理/区组/栋/单元/栏位列各 80px，实际内容（数字）仅需 40-50px，而品系/耳标颜色列需要更宽。
