# 实验结栏页 Review

**File:** `09-ExperimentEnd.html`
**Date:** 2026-06-22
**Product context:** Experiment R&D Platform — 实验结束（博士结栏确认 + 实验报告生成）

---

## 总评

实验结束页是实验生命周期的终点——博士在此确认所有阶段完成、查看最终合并数据、归档实验并生成报告。页面结构正确（实验信息头 + 流程节点 + 阶段 Tab + 处理组 Tab + 汇总卡片 + 实验总结表格），渐变统计卡片和确认按钮符合设计系统。但页面数据完全硬编码，缺少与前面阶段的联动，且实验报告生成仅有按钮无实现。

---

## 发现清单

### P0

**1. "生成最终实验报告"按钮无实现**

结栏确认区的核心 CTA——"生成最终实验报告"按钮没有绑定任何事件。这是博士做实验的核心目标（产出实验报告），演示时若点击无反应会严重影响可信度。

> **建议：** 至少跳转至 `10-AnalysisReport.html?expCode=XXX&tab=summary` 展示实验总结表。

**2. 结栏确认后缺少归档状态反馈**

点击"确认结栏"后无 feedback。应展示阶段状态转换：`已结束 → 已归档`，并锁定所有实验数据为只读。

> **建议：** `ElMessageBox.confirm('确认后将归档实验数据，不可再修改。是否继续？')`，确认后展示归档成功页面或跳转至报告页。

### P1

**3. 阶段 Tab 和处理组 Tab 数据硬编码**

阶段 Tab（01-DRB / 02-ZZB）的内容写死在 HTML 中，处理组 Tab 切换也没有动态加载对应数据。两个阶段的汇总指标完全相同——不合理（不同阶段应有不同数据）。

> **建议：** 从 URL 参数读取实验编号，通过 computed 按阶段和处理组动态展示对应的汇总数据。

**4. 实验总结表格缺少跨阶段对比列**

"各阶段数据对比"表格只有 01-DRB 一行示例数据（且显示"进行中"），没有 02-ZZB 的数据，无法进行真正的跨阶段对比。

> **建议：** 确保两个阶段的数据都展示在对比表格中，增加"变化率"列（如 ZZ 阶段 vs DRB 阶段的日增重变化 %）。

**5. 实验基本信息中的"实验周期"显示但缺少"实验结束日期"**

实验信息头部显示 `startDate` 和 `endDate`（计划日期），但没有"实际结束日期"。实验可能提前或延后结束。

> **建议：** 在计划结束日期旁增加"实际结束日期"字段，并在结栏时自动填充为当前日期。

### P2

**6. 缺少"导出完整实验数据包"功能**

博士可能需要导出所有原始数据+报表+附件为一个完整数据包用于存档或提交。

> **建议：** 增加"导出完整数据包（Excel + PDF + 附件）"按钮，原型阶段可用 toast 占位。

**7. 缺少实验结论/备注的录入区**

结栏前博士可能想写一段实验总结或备注（如"本次实验因XX原因提前终止"），当前页没有文本输入区。

> **建议：** 在结栏确认按钮上方增加"实验总结/备注"文本框。
