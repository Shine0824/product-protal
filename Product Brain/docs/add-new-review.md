# 新建实验页 Review

**File:** `01-Add_New.html`
**Date:** 2026-06-22
**Product context:** Experiment R&D Platform — 实验创建入口

---

## 总评

新建实验页面采用模态框 + 侧边产品说明面板的布局，表单字段基本覆盖了 Product Brain 中定义的实验创建所需信息。产品说明面板（"产品说明"）是一个有创意的设计——将需求备注与表单并置。但表单存在字段冗余、交互粗糙、缺少校验反馈等问题。

---

## 发现清单

### P0 · 演示前必须修复

**1. 提交无校验，仅 console.log 后关闭弹窗**

`handleSubmit()` 只执行 `console.log('提交表单', this.formData)` 然后关闭模态框。没有任何必填项校验、格式校验或提交确认。用户可以不填任何字段直接提交。

> **建议：** 实现必填项校验（标记了 `required` 的 8 个字段），校验失败高亮对应输入框并显示错误提示。提交成功后跳转到分组页面而非仅关闭弹窗。

**2. 实验类型选择后字段联动未实现**

Product Notes 中明确写了需求："当选择饲料营养类实验，才显示出新增的字段（生长阶段和试验阶段名称）"。但"生长阶段"和"试验阶段名称"字段始终可见且均标记为 `required`。"动物方向"仅有"猪"一个选项，"COO Area"仅有 COO 1/2 两个假选项。

> **建议：** 实现 `experimentType` watch → 控制 `growthStage` 和 `stages` 字段的可见性及必填状态。补全动物方向和 COO Area 的真实选项数据。

**3. 实验场选项使用代码缩写，难以理解**

实验场下拉为 `DY`、`NM`、`XY`，对应东营、内蒙古、襄阳。与概览页使用完整中文名称的惯例不一致。

> **建议：** 选项 label 使用完整中文名称（如"东营 (DY)"），value 保留代码。

### P1 · 建议优化

**4. 产品说明面板始终展开，遮挡表单**

产品说明面板在右侧 300px 固定宽度，展开时与表单并排。在小屏幕（< 1200px）上会挤压表单区域。面板内容是需求备注而非用户帮助，不应默认展示。

> **建议：** 默认收起产品说明面板（`notesExpanded: false`）。或将面板改为可拖拽宽度的侧边栏。

**5. 缺少实验编号预览**

Product Brain 定义了实验编号格式（如 `SWINE-NUR-XY-001-2026`），但表单中没有预览生成的编号。用户无法确认编号是否符合预期。

> **建议：** 在表单顶部增加实验编号预览行，根据动物方向 + 生长阶段 + 实验场 + 日期自动生成，支持手动修改。

**6. 缺少"取消"确认**

点击取消按钮或关闭按钮直接关闭弹窗，已填写的内容会丢失且无提示。

> **建议：** 表单有内容变更时，关闭前弹出确认对话框。

**7. 附件上传 action 为空**

`<el-upload action="" :auto-upload="false">` — action 为空字符串，虽设置了 `auto-upload="false"` 但缺少实际的上传逻辑和文件类型校验。

> **建议：** 至少实现前端文件类型过滤（pdf, xlsx, docx, pptx, jpg, png, mp4）和文件大小限制。

### P2 · 锦上添花

**8. 阶段名称缺少预设模板**

阶段名称默认 3 个空输入框，用户完全手工输入。Product Brain 中常用的阶段代码（DRB、ZZB 等）可以为新用户提供参考。

> **建议：** 增加阶段名称 placeholder 示例，或提供"常用阶段模板"下拉快速填充。

**9. 实验描述缺少富文本支持**

实验描述使用普通 textarea，但提示支持 pdf/xlsx/docx 等文件类型——暗示可能需要富文本或 Markdown。

> **建议：** 至少支持换行保留和多行输入，如需富文本替换为 `el-input type="textarea"` 并增加 Markdown 预览。

**10. 模态框缺少 loading 状态**

提交后没有 loading 反馈，`handleSubmit` 是同步的。真实场景需要异步提交。

> **建议：** 增加提交按钮的 loading 状态（`el-button :loading="submitting"`），模拟异步延迟。
