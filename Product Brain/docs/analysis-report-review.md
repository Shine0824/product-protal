# 报表分析页 Review

**File:** `10-AnalysisReport.html`
**Date:** 2026-06-22
**Product context:** Experiment R&D Platform — Web端报表分析

---

## 总评

报表分析页是原型中实现最完整的页面之一。8 个报表 Tab 均有实际数据和 UI 实现，Chart.js 集成到位（含自定义背景警示插件），日历热力图、趋势对比等可视化元素丰富。但搜索体验割裂（必须输入编号才能看报表）、图表销毁/重建逻辑有隐患、Tab 内容纯静态未联动数据。

---

## 发现清单

### P0 · 演示前必须修复

**1. 必须输入实验编号才能看到报表，初始为空状态**

页面加载时 `searchCode` 为空，显示"请输入实验编号查询报表数据"。从概览页跳转时可通过 URL 参数传入，但直接访问或刷新后丢失上下文。空状态引导文字有效但缺少快捷入口。

> **建议：** 增加"最近查看的实验"快捷入口，或从 URL 参数自动填充。空状态下展示实验选择器（下拉），而非仅文本提示。

**2. 图表在切换 Tab 时存在重复初始化风险**

`initDeathChart()` 在 watch 中调用（延迟 100ms），`initCharts()` 在 onMounted 中调用。如果用户在 Tab 间快速切换，可能创建多个 Chart 实例导致内存泄漏或渲染异常。`deathChart` 虽有 destroy 逻辑，但 `diarrheaChart` 和 `envChart` 没有。

> **建议：** 统一图表管理：在 Tab 切换时 destroy 当前图表实例，使用 `nextTick` 替代 `setTimeout`。所有图表实例存储为 ref 并在组件卸载时统一销毁。

**3. 4/8 报表 Tab 仅有静态数据，无交互**

日报表、阶段报表、实验总结表、称重明细表均为纯静态 HTML 表格数据，死淘、腹泻、药品疫苗、环控有图表和交互。体验割裂——用户切换 Tab 时期望一致。

> **建议：** 为所有 Tab 至少提供数据筛选（日期范围、处理组）和导出按钮。

### P1 · 建议优化

**4. 搜索栏与概览页完全相同但功能不同**

报表页的搜索栏（实验编号 + 实验场 + 实验状态）与概览页完全一致，但这里应搜索"要查看哪个实验的报表"，而非"过滤实验列表"。实验场和实验状态在此上下文中意义不明。

> **建议：** 将搜索栏改为实验选择器 + 日期范围选择器，更符合"查看报表"的语义。

**5. 报表缺少导出功能**

8 个报表 Tab 没有导出按钮。博士需要将报表导出为 Excel 进一步分析、或导出 PDF 分享——这是核心场景。

> **建议：** 在每个报表 Tab 右上角增加"导出 Excel"和"导出 PDF"按钮。原型阶段可用 `alert('导出功能开发中')` 占位。

**6. 死淘图表自定义插件逻辑脆弱**

`alertBackgroundPlugin` 硬编码了 `alertIndex = 3`（5/29日），在真实数据中需要数据驱动的高亮逻辑。`pointRadius` 的计算逻辑（`v >= 3 ? 8 : 6`）同样硬编码。

> **建议：** 将高亮规则提取为配置（如"当日死亡 ≥ 3 头时高亮"），从数据中动态计算 `alertIndex`。

**7. 药品疫苗日历热力图缺少月份导航**

日历始终显示当前月份。实验可能跨越多月，用户无法导航到历史月份查看。

> **建议：** 增加月份左右箭头导航，支持查看历史用药记录。

**8. 环控明细表缺少时间段筛选**

表格固定展示 2026/6/1 两天四次记录。真实场景中环控数据可能每天多次采集，需要按日期/时段筛选。

> **建议：** 增加日期选择器和时段筛选（上午/下午/全天）。

### P2 · 锦上添花

**9. 报表 Tab 顺序不符合使用频率**

日报表最常用应排第一（当前正确），但死淘明细（带高级图表）排第 4，在使用频率上可能高于阶段报表。

> **建议：** 收集用户反馈后调整顺序，或支持用户自定义 Tab 顺序。

**10. 实验信息条在页面上方但不可交互**

实验信息条展示编号、栋舍、阶段、更新时间，但这些字段都是静态文本。实验编号应可点击返回概览页。

> **建议：** 实验编号改为可点击链接，导航回概览页并保留搜索上下文。

**11. 缺少"打印"友好的样式**

报表页的核心场景包括打印存档，但表格列数多（日报表 28 列），打印时必然分页断裂。

> **建议：** 增加 `@media print` 样式，横向打印、重复表头、隐藏侧边栏和顶部导航。
