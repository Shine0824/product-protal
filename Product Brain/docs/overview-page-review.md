# 实验概览页 Review

**File:** `00-Experiment_overview.html` (parent directory)
**Date:** 2026-06-22
**Product:** Experiment R&D Platform · Web 端首页

---

## 总评

页面整体结构符合 Decision-001 的"概览优先"策略，四张统计卡片 + 实验列表 + 异常预警的三段式布局合理，与博士/CEO "一览全局"的核心需求对齐。但作为平台唯一确定的 Landing Page，当前版本在**可交互性、角色感知、异常可见性、数据上下文**四个维度有明显提升空间。

---

## 发现清单

### P0 · 演示前必须修复

**1. 报表入口实现不一致，5/8 用 alert() 占位**

报表下拉菜单中，前3项（日报表、阶段报表、实验总结表）跳转到 `10-AnalysisReport.html`，后5项（死淘、腹泻、药品疫苗、环控、称重）全部弹出 `alert()`。在演示中这会显得产品未完成。

> **建议：** 后5项也统一跳转到 Analysis Report 页，通过 `tab` 参数定位到对应报表 Tab。或在该页面为所有8种报表实现对应的 Tab 切换。

**2. 异常预警的"查看详情"用 alert() 展示原始数据**

异常预警是平台"智能"能力的核心体现之一（自动化检测 + 推送），但点击"查看详情"弹出的是浏览器原生 alert 对话框，显示 JSON 描述文本。演示中这会严重削弱产品感知。

> **建议：** 实现一个异常详情弹窗/抽屉，展示：异常指标趋势图、历史同类异常、建议处理方案、备注记录入口。至少用 `ElMessageBox` 替代 `alert()`。

**3. 异常预警区域位于页脚，关键预警不可见**

异常预警表格在页面最底部，屏幕高度不足时（尤其笔记本），预警信息完全不可见。但它是博士/CEO 决策的核心信息之一。顶部统计卡片的"异常数据记录"数字与底部表格之间没有视觉关联。

> **建议：** 在统计卡片区增加点击跳转（点击"异常数据记录"卡片滚动到预警区）。或考虑将预警区提升到实验列表上方，至少增加一个"有新预警"的视觉提示。

**4. 侧边栏导航完全不可交互**

除"实验概览"外，知识库、项目管理、实验场、猪实验数据、家禽实验数据均无任何交互响应。作为原型演示，这会引发"这些模块是否在计划中"的追问。

> **建议：** 至少为非活跃菜单项增加 tooltip 说明（如"功能开发中"），或将尚未实现的菜单项视觉上置灰。对于"报表分析"应实现跳转。

### P1 · 建议优化

**5. 统计卡片缺乏可点击性**

四张卡片是只读的。用户直觉会点击数字或卡片来查看详情（尤其"异常数据记录"和"今日新增记录"），但目前无响应。

> **建议：** 点击"实验总数"→ 展开状态分布细分（进行中/已完成/待开始各多少）。点击"异常数据记录"→ 滚动至下方预警表格。点击"今日新增记录"→ 跳转到当日记录详情。

**6. 缺少角色感知与个性化**

页面无论谁来都是完全相同的视图。但 PRODUCT-GUIDE 中定义了三个差异显著的角色：CEO 看全局资产、博士看实验细节、场长看数据质量。当前"李永霞"的头像只是静态展示。

> **建议：** 在用户头像旁增加角色标签（如"博士 · 数据产品负责人"）。未来可按角色调整统计卡片的默认展示（CEO 看汇总、博士看实验进度、场长看数据完整度）。原型阶段至少展示角色信息。

**7. 实验列表缺少默认排序与排序指示**

表格未指定默认排序规则，无排序箭头。CEO 看表时会困惑：按什么排序？最新的在前还是异常的在前面？

> **建议：** 默认按"异常等级"降序 + "记录完整度"升序排列，将最需要关注的实验排在最前。表头增加排序图标。

**8. 缺少数据刷新时间与当前日期上下文**

统计中"今日新增记录 156"、"较上周 +5%"依赖时间上下文，但页面没有显示当前日期或"数据更新时间"。用户无法判断数据的时效性。

> **建议：** 在统计卡片上方或右上角增加"数据更新时间：2026-06-22 09:30"及"今日日期"的展示。

**9. "当前进度"列语义不清晰**

实验列表中"当前进度"列的值混合了不同的语义：日常巡栏（动作）、阶段结束（状态点）、实验分组（配置步骤）、实验结束（终点）。用户难以快速判断下一步该做什么。

> **建议：** 将"当前进度"拆分为两组信息：(a) 阶段进度指示器（如 `01/04 阶段`）表示已完成阶段数；(b) 当前操作提示（如"待录入数据"/"待博士审核"）带有明确的行为指引。

**10. 实验编号列同时承载链接和信息展示，可发现性不足**

实验编号是绿色的粗体文字，hover 时没有下划线或颜色变化。用户可能不知道它是可点击的（当前进度列反而有下划线）。

> **建议：** 实验编号 hover 时增加下划线 + 颜色加深，或在行末增加一个"进入 →"操作列统一收口。

**11. 批量操作缺失**

页面没有任何批量操作能力。博士可能需要批量导出多个实验的报表，或批量修改实验状态。

> **建议：** 在实验列表上方增加多选 checkbox + 批量操作栏（批量导出、批量标记等）。原型阶段可先展示 UI 占位。

**12. "家禽实验数据"出现在侧边栏**

Product Brain 的 vision 和 scope 中未提及家禽。演示时会被追问"你们也做家禽吗？"。

> **建议：** 如果不在当前 scope，暂时隐藏或标记为"规划中"并置灰。避免演示时分心。

### P2 · 锦上添花

**13. 统计卡片缺少状态分布细分**

"实验总数 4"没有展示细分：几个进行中、几个已完成、几个待开始。CEO 关心的不仅是总数，更是分布。

> **建议：** 在"实验总数"卡片内增加小型状态分布条（如：`进行中 2 | 已完成 1 | 待开始 1`），或 hover 时展开。

**14. 侧边栏品牌区可增强**

侧边栏顶部只有 `🔬 实验研发` 四个字。可以增加平台版本号、组织名称，或一个收起/展开按钮以便在小屏幕上使用。

> **建议：** 增加一个小字号的副标题，如 "Experiment R&D Platform v1.0"。

**15. 响应式设计欠缺**

固定 220px 侧边栏 + 60px 顶部导航的布局在小于 1280px 的屏幕上会出现内容区过窄的问题。

> **建议：** 考虑侧边栏折叠模式（图标模式，60px 宽），顶部导航增加汉堡菜单。

**16. 无数据状态的引导**

如果实验列表或预警列表为空，需要一个引导性状态（而非空白表格）。当前预警区域有空状态设计（✅ 暂无数据异常预警），但实验列表没有对应的空状态。

> **建议：** 无实验时展示引导卡片："🎯 开始你的第一个实验 → 新建立项"。

**17. 搜索栏的"查询"按钮是无效操作**

`handleSearch()` 为空函数，过滤通过 `computed` 自动完成。但按钮的存在暗示用户必须点击才能搜索，这是一个交互歧义。

> **建议：** 将按钮文案改为"筛选"并保持可用（至少触发一个微小的反馈如 loading 闪烁），或完全移除按钮，改为输入即搜的交互模式。如果保留按钮，应确保 `handleSearch` 有实际逻辑（如记录搜索日志或触发数据刷新）。

**18. 缺少键盘快捷键支持**

作为博士/CEO 的高频操作页面，缺少键盘导航支持。

> **建议：** 支持 `/` 聚焦搜索框，`Esc` 清除筛选等。
