# 日常巡栏页 Review

**File:** `03-DailyRecords.html`
**Date:** 2026-06-22
**Product context:** Experiment R&D Platform — App端/Web端 日常巡栏数据录入（核心操作页面）

---

## 总评

日常巡栏页是整个平台中交互最丰富的页面——Tab 切换数据维度、折叠区展开/收起、数据表格内联编辑、查询弹窗。它正确地将 6 个数据维度组织为 Tab（死淘/采食/环控/药品疫苗/腹泻/称重），用折叠区展示不同栏位的记录。但编辑对话框在弹窗内实现（而非内联编辑），"查询"功能使用独立模态框打断了用户流程。

---

## 发现清单

### P0

**1. 数据记录缺少保存确认和同步反馈**

编辑完成后关闭弹窗，数据在本地 `currentModuleRecords` 中更新，但没有"保存成功"的反馈，也没有体现"实时同步到云端"（Product Brain 的核心价值之一）。

> **建议：** 保存后使用 `ElMessage.success('记录已保存并同步至云端')` 模拟同步反馈。在页面顶部或底部增加"上次同步时间"指示器。

**2. 死淘记录的"累计"字段 disabled 但计算逻辑不可见**

累计死亡数量/重量/淘汰数量等字段为 `disabled`，标注"自动计算"。但用户无法看到计算是否正确，也无法手动修正。

> **建议：** 在累计字段旁增加 tooltip 图标，hover 显示计算公式和所依赖的历史记录条数。

### P1

**3. 查询弹窗使用独立模态框，打断录入流程**

"查询"按钮打开一个固定定位的模态框（`query-modal`），覆盖在录入表单之上。用户在录入过程中想查询历史数据时需要中断当前操作。

> **建议：** 将查询改为侧边抽屉（`el-drawer`）或底部展开面板，允许用户在不离开录入界面的情况下查看历史数据。

**4. 栏位筛选器（pen-filter）与 Tab 切换的交互不明确**

栏位下拉筛选在表单上方，切换栏位后 Tab 下的数据会变化。但用户可能不理解两者之间的关联。

> **建议：** 在栏位筛选器旁增加标签说明："选择栏位以筛选该栏位的记录数据"，或在切换栏位时 Tab 内容有 loading 过渡。

**5. 缺少"今天已记录"vs"今天未记录"的视觉区分**

用户进入页面后无法立即知道"今天哪些栏位已经巡栏过了"。需要手动切换栏位/日期才能发现。

> **建议：** 在栏位列表中增加完成状态标记（✓ 已记录 / ○ 待记录），帮助场长快速识别遗漏。

### P2

**6. 折叠区展开/收起缺少动画过渡**

`collapse-content` 使用 `v-if`（直接插入/移除 DOM），无过渡动画。大量数据切换时体验生硬。

> **建议：** 使用 Vue `<Transition>` 包裹折叠内容，增加 `max-height` 过渡动画。

**7. 编辑对话框中的"删除"按钮无二次确认**

每条记录行末的删除按钮直接删除，无确认步骤。死淘/采食数据一旦误删可能影响实验结论。

> **建议：** 使用 `ElMessageBox.confirm` 二次确认删除操作。
